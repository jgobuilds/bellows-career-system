## Conventions (the system reads these)
### Capitalization
- **Capitalize** when it's a name or a label: section headers, skills-list items, product names.
- **Lowercase** when it's a generic concept in prose: {e.g. "data governance" mid-sentence}.
- ATS keyword matching is case-insensitive — there is no benefit to mimicking a posting's capitalization.

### Never do this
- No invented numbers. No scale overreach. No skills you haven't actually used.
- Expired certifications are always labeled prior.
- If a claim isn't traceable to this file, it does not go on a resume.
